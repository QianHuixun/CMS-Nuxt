# 沉浸式编排联调验证清单

## 目标

验证沉浸式编排在真实登录态下的“保存 - 刷新 - 回显”链路是否正常。

## 前置条件

- `MySQL` 已启动
- `Redis` 已启动
- `chunyu-cms-web` 已启动，`http://localhost:3000` 可访问
- `chunyu-cms-admin` 已启动，`http://localhost:4000` 可访问
- 可正常登录管理端

## 环境冒烟

- [x] `GET /api/admin/captchaImage` 返回 `200`
- [x] `POST /api/admin/logout` 返回 `200`

## 用例清单

### 1. 场景新建与刷新回显

步骤：

1. 进入“活动管理 / 沉浸式编辑”
2. 新建一个场景，填写名称、描述、缩略图
3. 保存场景
4. 刷新页面
5. 观察左侧场景列表

预期：

- 新建场景仍然存在
- 名称、描述、缩略图正确回显

### 2. 可视化编排文本块保存与刷新回显

步骤：

1. 选中一个场景
2. 添加文本块
3. 编辑文本内容
4. 点击“保存”
5. 刷新页面并重新进入该场景

预期：

- 文本块仍存在
- 文本内容正确回显

### 3. 可视化编排图片块保存与刷新回显

步骤：

1. 添加图片块
2. 上传图片或填写图片 URL
3. 点击“保存”
4. 刷新页面并重新进入该场景

预期：

- 图片块仍存在
- 图片地址、宽度、圆角等属性正确回显

### 4. 可视化编排视频块保存与刷新回显

步骤：

1. 添加视频块
2. 上传视频
3. 等待自动封面生成
4. 点击“保存”
5. 刷新页面并重新进入该场景

预期：

- 视频块仍存在
- 视频地址保存为服务端 `/uploads/...` 路径
- 封面地址保存为服务端 `/uploads/...` 路径
- 不再出现 `content_blocks too long` 报错

### 5. block 删除后保存与刷新回显

步骤：

1. 在同一场景下创建多个 block
2. 删除其中一个 block
3. 点击“保存”
4. 刷新页面

预期：

- 被删除 block 不再出现
- 其他 block 排序正常

### 6. block 排序后保存与刷新回显

步骤：

1. 创建多个 block
2. 使用上移、下移或拖拽调整顺序
3. 点击“保存”
4. 刷新页面

预期：

- block 顺序与保存前一致

### 7. 富文本模式保存与刷新回显

步骤：

1. 切换到“富文本模式”
2. 输入一段富文本内容
3. 点击“保存”
4. 刷新页面

预期：

- 富文本内容正确回显
- 不影响场景列表加载

### 8. 后端重启后的页面恢复

步骤：

1. 登录后台并进入沉浸式编辑页
2. 重启 `chunyu-cms-web`
3. 刷新页面

预期：

- 不再因为 `/logout 503` 卡成白屏
- 最差情况应回到登录页，而不是停留空白页
