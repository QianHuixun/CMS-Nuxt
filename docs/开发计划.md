# 开发计划

本文基于 `docs/项目需求分析.md` 和 `docs/数据库字段规划.md` 输出，用于指导第一阶段后台管理、接口、数据表和前端展示联调开发。

## 1. 开发目标

第一阶段以“先可上线、后做增强”为原则，优先完成 6 个一级模块的 MVP 闭环：

- 系统基础配置管理
- 资源导航管理
- 人才与对应科研管理
- 论文著作与成果管理
- 自动化辅助功能
- 活动管理

核心目标：

- 后台菜单和页面全部可访问。
- 关键业务数据可新增、编辑、删除、查询和展示。
- 人才与论文、课题、专利的关联关系可保存、回显和修改。
- 图片、文件、视频上传优先复用现有组件。
- 自动化能力先做基础版，不做复杂算法平台化。
- 沉浸式编辑第一阶段采用富文本方式落地。

## 2. 当前项目落点

当前仓库为前后端分目录结构，开发落点如下：

- 后台前端：`chunyu-cms-admin/src`
- 后台页面：`chunyu-cms-admin/src/views`
- 后台 API 封装：`chunyu-cms-admin/src/api`
- 服务端：`chunyu-cms-web/server`
- 服务端接口：`chunyu-cms-web/server/api/admin`
- 服务层：`chunyu-cms-web/server/services/admin`
- Drizzle schema：`chunyu-cms-web/server/db/schema`
- Drizzle migration：`chunyu-cms-web/server/db/migrations`

注意：需求分析文档中提到的 `server`、`src/views` 为逻辑路径，实际开发时需要映射到上述项目目录。

## 3. 总体开发阶段

### 阶段 0：需求与菜单口径确认

目标：

- 将需求文档、菜单、占位页面和数据表规划统一到同一口径。

开发内容：

- 校验 6 个父菜单和子菜单是否与 `docs/项目需求分析.md` 一致。
- 校验菜单 migration 中的路由、权限标识、排序、组件路径。
- 确认“人才管理”“专家库管理”“课题时空管理”的命名和页面映射。
- 确认第二阶段内容不混入第一阶段开发范围。

完成标准：

- 6 个父菜单和全部子菜单后台可见。
- 所有菜单页面可访问，不出现 404。
- 第一阶段 MVP 范围清晰。

### 阶段 1：数据库与 Schema 建设

目标：

- 先建立能支撑 MVP 闭环的数据结构。

第一阶段建表清单：

1. `system_nav_config`
2. `screen_saver_config`
3. `screen_saver_media`
4. `resource_navigation`
5. `talent`
6. `paper`
7. `topic`
8. `patent`
9. `talent_paper`
10. `talent_topic`
11. `talent_patent`
12. `book_simulation`
13. `book_simulation_page`
14. `import_batch`
15. `paper_keyword`
16. `activity`

暂缓建表：

- `activity_content`：第二阶段模块化沉浸式编辑使用。
- `activity_headline`：第二阶段多栏目、多时间段头条使用。
- `statistics_snapshot`：第二阶段统计快照缓存使用。

开发内容：

- 在 `chunyu-cms-web/server/db/schema` 下按业务域新增 schema。
- 生成并检查 Drizzle migration。
- 补齐通用字段：`remark`、`create_by`、`create_time`、`update_by`、`update_time`。
- 统一状态字段：`status`，`0` 正常/启用，`1` 停用/隐藏。
- 统一逻辑删除字段：`del_flag`，`0` 未删除，`1` 已删除。
- 统一排序字段：`sort`，数值越小越靠前。

完成标准：

- migration 可正常执行。
- schema 可被业务 service 正常引用。
- 多对多关系表能支撑人才与成果关联。

### 阶段 2：后端接口与服务层建设

目标：

- 为每个模块建立统一的后台管理接口。

建议接口目录：

- `chunyu-cms-web/server/api/admin/systemBaseConfig`
- `chunyu-cms-web/server/api/admin/resourceNavigation`
- `chunyu-cms-web/server/api/admin/talentResearch`
- `chunyu-cms-web/server/api/admin/paperAchievement`
- `chunyu-cms-web/server/api/admin/automationTools`
- `chunyu-cms-web/server/api/admin/activityManagement`

建议服务目录：

- `chunyu-cms-web/server/services/admin/systemBaseConfig`
- `chunyu-cms-web/server/services/admin/resourceNavigation`
- `chunyu-cms-web/server/services/admin/talentResearch`
- `chunyu-cms-web/server/services/admin/paperAchievement`
- `chunyu-cms-web/server/services/admin/automationTools`
- `chunyu-cms-web/server/services/admin/activityManagement`

通用接口能力：

- 分页列表
- 详情查询
- 新增
- 编辑
- 逻辑删除
- 状态切换
- 排序更新
- 下拉选项接口

专项接口能力：

- 人才保存时同步维护 `talent_paper`、`talent_topic`、`talent_patent`。
- 论文、课题、专利提供可搜索下拉接口，供人才管理选择关联。
- 论文、课题、专利导入后记录 `import_batch`。
- 词云提供关键词列表、权重更新和按论文筛选能力。
- 图表生成基于 `paper.publish_year` 动态聚合，不单独落统计表。
- 活动头条第一阶段直接使用 `activity.is_headline` 和 `activity.headline_sort`。

完成标准：

- 每个模块具备后台 CRUD 基础接口。
- 接口返回结构与现有后台项目风格一致。
- 业务删除均为逻辑删除。
- 关联关系保存、查询和更新可闭环。

### 阶段 3：后台前端页面开发

目标：

- 将占位页面替换为真实管理页面，并接入接口。

页面开发目录：

- `chunyu-cms-admin/src/views/systemBaseConfig`
- `chunyu-cms-admin/src/views/resource/navigation`
- `chunyu-cms-admin/src/views/talentResearch`
- `chunyu-cms-admin/src/views/paperAchievement`
- `chunyu-cms-admin/src/views/automationTools`
- `chunyu-cms-admin/src/views/activityManagement`

API 封装目录：

- `chunyu-cms-admin/src/api/systemBaseConfig`
- `chunyu-cms-admin/src/api/resourceNavigation`
- `chunyu-cms-admin/src/api/talentResearch`
- `chunyu-cms-admin/src/api/paperAchievement`
- `chunyu-cms-admin/src/api/automationTools`
- `chunyu-cms-admin/src/api/activityManagement`

可复用组件：

- 图片上传：`ImageUpload`
- 文件上传：`FileUpload`
- 视频上传：`VideoUpload`
- 富文本编辑：`RichTextEditor`
- 头像裁剪：复用现有用户头像裁剪能力

通用页面结构：

- 查询区
- 表格区
- 操作区
- 新增/编辑弹窗或独立表单页
- 状态开关
- 排序输入或上移下移
- 删除确认

完成标准：

- 页面可完成完整新增、编辑、删除、查询流程。
- 上传组件和表单回显正常。
- 列表刷新、分页、筛选、排序状态稳定。

## 4. 模块开发计划

### 4.1 系统基础配置管理

子菜单：

- 导航配置
- 屏保配置

数据表：

- `system_nav_config`
- `screen_saver_config`
- `screen_saver_media`

开发内容：

- 导航配置支持首页入口名称、副标题、简介、跳转链接、链接类型、图标、封面、排序、启用状态。
- 屏保配置支持配置名称、无人操作触发秒数、播放模式、循环状态、启用状态。
- 屏保素材支持图片和视频，包含素材地址、视频封面、标题、时长、排序、状态。
- 支持当前配置预览。

验收标准：

- 导航配置可新增、编辑、删除、启停和排序。
- 屏保配置可维护基础参数和素材列表。
- 图片、视频上传和预览可用。

### 4.2 资源导航管理

子菜单：

- 资源录入
- 卡片管理

数据表：

- `resource_navigation`

开发内容：

- 资源录入支持资源名称、简介、封面图、访问链接、演示视频、视频封面、链接类型、打开方式、排序、显示状态。
- 卡片管理支持列表、删除、显示/隐藏和排序调整。
- 第一阶段排序可使用排序值或上移下移，不强制做拖拽。

验收标准：

- 资源可录入、编辑、删除。
- 资源卡片可控制显示隐藏和排序。
- 封面图、演示视频和视频封面可正常保存与回显。

### 4.3 论文著作与成果管理

子菜单：

- 数据导入
- 书籍仿真录入

数据表：

- `paper`
- `topic`
- `patent`
- `book_simulation`
- `book_simulation_page`
- `import_batch`

开发内容：

- 数据导入支持论文、专利、课题三类。
- 导入批次写入 `import_batch`，记录原始文件、总数、成功数、失败数、状态和错误摘要。
- 论文支持基础信息、摘要、关键词、PDF 全文、精选标记、精选排序。
- 专利支持基础信息、演示视频、视频封面、视频时长。
- 课题支持基础信息、负责人、起止年份/日期、地点、经纬度。
- 书籍仿真支持基础信息、封面、封底和核心章节图片集。

验收标准：

- 论文、专利、课题可导入或录入。
- 导入结果可分页展示。
- 论文精选可标记和排序。
- 专利视频上传、封面帧或手动封面可用。
- 书籍封面、封底和页面图片可上传、排序和回显。

### 4.4 人才与对应科研管理

子菜单：

- 人才管理

数据表：

- `talent`
- `talent_paper`
- `talent_topic`
- `talent_patent`
- 依赖 `paper`
- 依赖 `topic`
- 依赖 `patent`

开发内容：

- 人才基础信息 CRUD。
- 支持高清照片上传。
- 支持照片裁剪。
- 支持关联已存在的论文、课题、专利。
- 关联选择使用可搜索下拉多选。
- 保存人才时同步维护三张关联表。
- 编辑人才时正确回显已关联成果。
- 人才页面不直接新增论文、课题、专利。

验收标准：

- 人才可新增、编辑、删除、启停和排序。
- 人才照片上传和裁剪可用。
- 论文、课题、专利关联可保存、回显、修改。
- 关联数据删除或停用后，页面有合理兼容处理。

### 4.5 自动化辅助功能

子菜单：

- 词云
- 图表生成

数据表：

- `paper_keyword`
- 依赖 `paper`

开发内容：

- 词云支持单篇论文关键词维护。
- 词云支持全部收录论文关键词展示。
- 支持人工调整关键词权重。
- 关键词来源支持人工、摘要提取、导入。
- 图表生成支持按年度统计论文数量并生成折线图。
- 第一阶段统计结果动态查询，不落 `statistics_snapshot`。

验收标准：

- 词云可基于真实论文关键词生成。
- 关键词权重可编辑并影响展示。
- 年度论文数量图表可按 `publish_year` 正确聚合。

### 4.6 活动管理

子菜单：

- 沉浸式编辑
- 头条精选

数据表：

- `activity`

开发内容：

- 活动基础信息 CRUD。
- 沉浸式编辑第一阶段采用富文本，内容保存到 `activity.content_html`。
- 支持图文混排，支持图片、表格、视频等富文本内容维护。
- 支持封面图、活动日期、地点、摘要、发布状态。
- 头条精选使用 `is_headline` 和 `headline_sort` 实现。
- 支持活动置顶、排序、上下架状态。

验收标准：

- 活动内容可新增、编辑、预览和发布。
- 富文本内容可保存和回显。
- 头条精选可开启、关闭和排序。
- 活动上下架状态可控制前端展示。

## 5. 推荐开发顺序

建议按以下顺序推进：

1. 菜单、路由、占位页与权限标识校验。
2. 第一阶段数据库 schema 和 migration。
3. 后端 service 与通用 CRUD 接口模板。
4. 系统基础配置管理。
5. 资源导航管理。
6. 论文著作与成果管理。
7. 人才与对应科研管理。
8. 自动化辅助功能。
9. 活动管理。
10. 后台、接口、前台展示联调。
11. 验收测试和发布准备。

排序原因：

- 系统配置和资源导航依赖较少，适合优先完成可见成果。
- 论文、课题、专利是人才关联和自动化功能的数据基础，需先完成。
- 自动化辅助依赖论文数据沉淀，应放在成果数据之后。
- 活动管理相对独立，但依赖上传和富文本链路稳定，适合后置开发。

## 6. 权限建议

每个子菜单至少配置以下权限：

- `list`
- `query`
- `add`
- `edit`
- `remove`
- `export` 或 `import`，仅导入导出页面需要

可选独立权限：

- `status`：状态切换
- `sort`：排序更新
- `featured`：精选标记
- `headline`：头条精选
- `upload`：上传操作

## 7. 联调与测试计划

后端测试重点：

- migration 执行成功。
- 每个 CRUD 接口参数校验、分页、逻辑删除、状态切换正确。
- 人才关联成果事务一致性正确。
- 导入失败时 `import_batch` 错误信息可追踪。
- 图表聚合结果与数据库真实数据一致。

后台前端测试重点：

- 菜单跳转无 404。
- 表单必填校验和默认值正确。
- 上传、预览、删除、回显链路正确。
- 多选关联数据保存和回显正确。
- 列表分页、查询、重置、排序、状态切换正确。

端到端验收重点：

- 导航配置保存后前端可读取展示。
- 资源导航卡片可展示、隐藏和排序。
- 论文、课题、专利数据可被人才管理关联。
- 词云和图表可基于真实论文数据生成。
- 活动发布后头条精选可展示。

## 8. 风险与处理策略

- 菜单命名和需求文档存在差异：先统一口径，再改页面和接口命名。
- 人才关联成果依赖论文、课题、专利主数据：先做成果主数据，再做人才关联。
- 视频自动压缩成本较高：第一阶段只做上传、MP4 校验、封面帧或手动封面。
- 词云分词方案不确定：第一阶段优先基于人工关键词和导入关键词。
- 沉浸式编辑完整模块化能力复杂：第一阶段用富文本，模块化内容表延后。
- 前台仿真翻页效果可能超出后台录入范围：第一阶段先完成后台数据维护，前台效果单独联调。

## 9. 第一阶段验收标准

- 6 个父菜单和全部子菜单在后台可访问。
- 每个模块 MVP 至少完成一条从录入到展示的闭环。
- 论文、课题、专利可录入或导入，并参与人才关联。
- 人才与论文、课题、专利关联可新增、回显和修改。
- 文件、图片、视频上传链路稳定可用。
- 词云和图表能基于真实论文数据产出结果。
- 活动内容可编辑、发布，并具备头条精选能力。
- 所有删除均采用逻辑删除，避免误删造成前台数据缺失。
