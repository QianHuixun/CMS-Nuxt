# 数据库字段规划

本文基于 `docs/项目需求分析.md` 和 `docs/项目需求.md` 规划第一阶段数据表字段，用于后续创建 Drizzle schema 和 SQL migration。

## 设计约定

- 表名使用小写下划线：如 `system_nav_config`。
- 主键使用 `表名语义_id`：如 `talent_id`、`paper_id`。
- 状态字段统一使用 `status`：`0` 正常/启用，`1` 停用/隐藏。
- 逻辑删除字段统一使用 `del_flag`：`0` 未删除，`1` 已删除。
- 排序字段统一使用 `sort`，数值越小越靠前。
- 文件字段优先存储可访问 URL 或相对路径，不直接存储二进制内容。
- 业务主表默认补齐现有通用字段：`remark`、`create_by`、`create_time`、`update_by`、`update_time`。
- 多对多关系使用中间表，不在主表中用字符串冗余存储 ID 列表。
