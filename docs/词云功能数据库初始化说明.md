# 词云功能数据库初始化说明

## 适用范围

本说明仅用于本期“自动化辅助 > 词云”最小可用版本。

本期仅新增：

- `paper`
- `paper_keyword`

不包含完整论文管理能力，不包含论文新增/编辑页面。

## 已新增 migration

- `chunyu-cms-web/server/db/migrations/0017_create_paper_word_cloud_tables.sql`

该 migration 包含：

- 创建 `paper` 表
- 创建 `paper_keyword` 表
- 初始化最小论文测试数据
- 初始化最小关键词明细数据

## 初始化执行方式

按项目既有数据库迁移流程执行即可。

如果当前环境使用 Drizzle migration，请确保：

1. 数据库连接配置正确
2. `0017_create_paper_word_cloud_tables.sql` 已被纳入迁移执行顺序
3. 执行后确认 `paper` 与 `paper_keyword` 两张表已存在

## 初始化完成后建议检查

建议执行以下检查：

1. `paper` 表是否有 3 条初始化数据
2. `paper_keyword` 表是否已有对应关键词明细
3. 登录后台后，`自动化辅助 > 词云` 页面是否能直接展示“全部论文”词云
4. 切换到单篇论文时，是否能拉到论文列表与关键词明细

## 当前接口依赖

词云页面依赖以下最小接口：

- `GET /api/admin/paper/list`
- `GET /api/admin/paper/:id`
- `GET /api/admin/paper/:id/keywords`
- `PUT /api/admin/paper/:id/keywords`
- `GET /api/admin/paper/keywords/cloud`

## 数据规则说明

### 单篇关键词读取规则

优先级如下：

1. 优先读取 `paper_keyword`
2. 若无明细，则从 `paper.keywords` 拆分
3. 拆分后的默认 `weight = 1`

### 全库词云聚合规则

- 仅按 `paper_keyword.keyword` 聚合
- 聚合方式为 `weight` 求和

说明：

- 保存权重只写 `paper_keyword`
- 不回写 `paper.keywords`

## 已知限制

- 如果某篇论文只有 `paper.keywords`，但还没有生成 `paper_keyword` 明细，则该论文不会自动参与全库聚合
- 本期不提供论文管理页面来维护基础论文数据；新增测试论文需要通过 SQL、seed 或 migration 处理
