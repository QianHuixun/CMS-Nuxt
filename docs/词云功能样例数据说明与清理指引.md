# 词云功能样例数据说明与清理指引

## 说明

当前词云功能使用的 3 篇论文与对应关键词，属于本期为了支持开发、联调、测试和验收而初始化的**样例数据**，不是正式业务数据。

这些样例数据来自：

- `chunyu-cms-web/server/db/migrations/0017_create_paper_word_cloud_tables.sql`

## 为什么会有这些数据

本期词云功能只交付最小可用版本：

- 做词云页面
- 做最小 `paper` / `paper_keyword` 数据能力
- 不做完整论文管理 CRUD

由于当前阶段还没有真实论文主数据，为了保证：

- 页面可正常打开
- 全部论文词云可展示
- 单篇论文模式可切换
- 权重保存与回显可验证

因此在 migration 中插入了最小样例数据。

## 当前样例数据的影响

只要这些样例数据还存在数据库中，系统就会继续使用它们：

- `全部论文` 模式会把样例数据一起参与聚合
- `单篇论文` 下拉里也会继续显示这些样例论文

这意味着：

- 后续导入真实论文数据后，系统**不会自动忽略**这些样例数据
- 如果希望后续环境只显示真实论文，需手动清理这些样例数据

## 建议使用方式

### 当前开发 / 测试环境

建议保留样例数据，用于：

- 功能联调
- 演示
- 测试
- 验收

### 后续真实数据环境

当真实论文数据准备完成后，建议：

1. 保留 `paper` / `paper_keyword` 表结构
2. 删除当前 3 篇样例论文及其关键词明细
3. 再导入真实论文数据

## 当前样例论文

当前初始化的 3 篇样例论文为：

1. `天回医简整理与川派医学文献数字化研究`
2. `中医脉诊语义标注与智能分析方法研究`
3. `针灸古籍图像识别与术语抽取实验`

## 清理样例数据 SQL

在确认要切换为真实论文数据前，可执行以下 SQL 清理样例数据：

```sql
DELETE pk
FROM paper_keyword pk
INNER JOIN paper p ON p.paper_id = pk.paper_id
WHERE p.title IN (
  '天回医简整理与川派医学文献数字化研究',
  '中医脉诊语义标注与智能分析方法研究',
  '针灸古籍图像识别与术语抽取实验'
);

DELETE FROM paper
WHERE title IN (
  '天回医简整理与川派医学文献数字化研究',
  '中医脉诊语义标注与智能分析方法研究',
  '针灸古籍图像识别与术语抽取实验'
);
```

## 清理后建议检查

执行清理后，建议检查：

1. `paper` 表中是否已移除上述 3 篇样例论文
2. `paper_keyword` 表中是否已移除对应关键词明细
3. 词云页面下拉中是否不再出现样例论文
4. 全部论文词云是否只基于真实数据展示

## 注意事项

- 不要删除 `paper` / `paper_keyword` 表，只删除样例数据
- 如果真实数据尚未导入，删除后词云页面可能为空，这是正常现象
- 建议在生产上线前再次确认是否仍存在样例论文
